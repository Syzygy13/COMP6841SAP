Timing Attacks

Timing attacks exploit the time it takes for a system to process inputs.


Attackers gain information when a successful query takes longer to execute than an unsuccessful one.


These are commonly found in: Password checks and Any program that takes a long time to compute.


' OR IF(SUBSTRING(password, 1, 1) = 's', SLEEP(5), 0) --


This input checks if the first character of the password is 's'.


If it is, the program sleeps for 5 seconds, indicating a correct guess.


If not, the operation runs normally with no delay.


This is much trickier to defend against however:


Developers can implement identical time comparisons for all operations using wait functions.


This helps ensure that both successful and unsuccessful operations take the same amount of time to complete.
