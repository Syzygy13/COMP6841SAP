Piggy-Backed Attacks

Piggy-backed attacks are similar to UNION query attacks. They use a delimiter to end a legitimate query and then add malicious commands.


Any additional SQL statements that follow the original query will be executed. Attackers can use multiple SQL command words to complete the injection.


'; DELETE FROM users WHERE '2' = '2'; SELECT * FROM users; --


The delimiter ';' ends the initial query early.


DELETE FROM users WHERE '2' = '2'; command deletes all records in the users table because the condition '2' = '2' is always true.


SELECT * FROM users; command attempts to retrieve everything from the users table. It should find nothing.


The -- at the end comments out the rest of the original SQL query.


To defend:


Parameterised queries are the most effective defense. By this point you can see they are used almost everywhere.


Developers can manually sanitise inputs use to filter out dangerous commands.


Disabling multiple queries in the database can block anything after the delimiter.
