Tautology Attacks

Tautology in logic means a statement that is true based on its logical structure. In SQL injections, an example is '1' = '1', which is always true because it compares two identical values.


On a login page, a tautology statement cannot run alone. The previous statement must be closed properly.


SELECT * FROM users WHERE username='input1' AND password='input2';


If you use '1' = '1' alone, it breaks the syntax. To make it work, you can use ' OR '1' = '1' --.


' OR '1' = '1' -- closes the original WHERE condition and adds the OR '1' = '1', which is always true.


Everything after -- is treated as a comment and ignored, making the password irrelevant and granting access.


Tautology attacks often target WHERE statements, which are common in login queries that search for matching accounts.


To defend:


Use parameterised queries to properly handle user input.


Restrict user input to exclude characters like single quotes.
