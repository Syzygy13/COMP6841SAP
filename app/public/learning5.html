Blind SQL Injection Attacks

Blind SQL injection is a method where an attacker asks true or false questions to the database to infer information. Similar to 20 Questions.


This attack works by manipulating queries to see different responses based on true or false conditions.


SELECT * FROM users WHERE username = '$username' AND password = '$password';


The attacker can close the original query and inject their own to check conditions.


For instance, they can use SUBSTRING to guess the password character by character without revealing errors.


Data extraction will occur one at a time making this a very tedious process.


Having no information from the beginning makes this purely a guessing game to even get started.


To defend:


Use parameterised queries to prevent most injections.


Manually sanitise user inputs to remove harmful characters/statements.


Do not expose error handling or detailed error messages to outsiders.
