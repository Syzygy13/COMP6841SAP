Union Queries Attacks

UNION attacks use the UNION SQL command to access the same or a different table. Attackers often target other tables they have information on, such as attributes, size, or existing content.


To execute this attack, the injected statement must first stop the original query, similar to the tautology attack.


' UNION SELECT attribute1, attribute2 FROM table_name --


The UNION keyword combines two queries, executing the developer's original query first and then the attacker's injected query.


The number of columns in both queries must match. If the original query has two columns, the attacker's query must have two as well.


The data types in the columns must be compatible with what the developer originally specified.


To defend:


Parameterised queries are highly effective as they restrict user inputs.


Manually sanitising inputs, such as removing instances of the UNION keyword also helps.


Limit error messages to prevent attackers from gathering useful information.
